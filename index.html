Dependency Injection: Aurelia


import {inject} from 'aurelia-framework';
import {UserService} from 'users/user-service';

@inject(UserService)
export class Users {
	constructor(userService) {
		this.userService = userService;
	}
	activate() {
		return this.userService.getAll()
			.then( users => this.users = users );
	}
}

Data Binding: The Aurelia Way

  • Append .bind to any html attribute
  • Defaults to two-way for input, one-way for everthing else
  • Want more control? ".one-way", ".two-way", ".one-time"
  • Adaptive Binding used to pick optimal observation strategy and minimize dirty checking

Angular Directive


angular.module('myApp').directive('myDatepicker', function () {
    return {
      templateUrl: 'views/mydatepicker.html',
      restrict: 'E',
      scope: { data: '=' },
			controller: function($scope, $element) {
				// Directive-specific logic
			},
      link: function postLink(scope, element) {
        // DOM manipulation here
      }
    };
  });

Configuration to specify where the view is located

Isolated scope, otherwise defaults to inheriting from parent (bad for re-usability)

Controller function runs first (no data binding yet)

Link function (aka postLink) runs last (template linked to scope)

More details on Directive Lifecycle

Directive: Datepicker


link: function(scope, element) {
	// Initialize datepicker
	$('input').datepicker();

	// Try to update data binding?
	$('input').datepicker().on('changeDate', function(e) {
		scope.data = e.date;
	});
}

Data binding will NOT update because event happened outside of Angular context

Directive: Datepicker


link: function(scope, element) {
	// Initialize datepicker
	$('input').datepicker();

	// Update data binding
	$('input').datepicker().on('changeDate', function(e) {
		scope.$apply(function () {
			scope.data = e.date;
		});
	});
}

Use scope.$apply to trigger a digest cycle

Leaky abstraction, requires knowledge of framework's internals

Directive: Watch

Populate Datepicker from Model


link: function(scope, element) {
	scope.$watch('data', function(newVal) {
    $('input').datepicker('setDate', scope.data);
  });
}

scope.$watch to add to list of watchers evaluated in digest cycle

Aurelia Custom Element

  • Works seamlessly with data binding
  • Finds view template by convention
  • Clearly named lifecycle methods

Custom Element: Datepicker


import {inject, customElement, bindable} from 'aurelia-framework';
import 'bootstrap-datepicker/dist/js/bootstrap-datepicker';

@customElement('my-datepicker')
@inject(Element)
export class DatePicker {
  @bindable data;
  constructor(element) { this.element = element; }
	bind() { // DOM manipulation here... }
  dataChanged(newVal) { // update datepicker here... }
}

Custom Element: Data Binding


bind() {
	// Find input element
	this.selector = $(this.element).find('.date');

	// Initialize datepicker
	this.selector.datepicker();

	// Update bound data on selection
	this.selector.datepicker().on('changeDate', (e) => {
		this.data = e.date;
	});

	// Prepopulate with bound data
	if (this.data) {
		this.selector.datepicker('setDate', this.data);
	}
}

Custom Element: Update


dataChanged(newVal, oldVal) {
	// this.data is automatically bound to newVal
	this.selector.datepicker('setDate', this.data);
}
